# APIExtractor

APIExtractor 是一个面向渗透测试、SRC 漏洞挖掘和接口安全评估场景的自动化辅助工具。项目目标是减少从网页中手动提取接口、拼接请求、批量测试接口可访问性的重复工作，帮助安全测试人员快速发现潜在的未授权访问、敏感接口暴露和异常接口响应等问题。

## 项目背景

在实际渗透测试和 SRC 挖掘过程中，API 安全通常是一个高价值方向。很多 Web 应用会在前端页面、JavaScript 文件、XHR/fetch 请求、静态配置或路由信息中暴露大量接口地址。

传统流程通常包括：

1. 使用浏览器开发者工具或浏览器插件收集接口。
2. 手动整理接口路径、参数、请求方法和请求头。
3. 手动复制到 Burp Suite、Postman、curl 或脚本中重新发包。
4. 根据状态码、响应长度和响应内容判断是否存在未授权访问风险。

这个过程重复性高、效率低，并且容易遗漏接口。APIExtractor 希望将上述流程自动化：从目标网页中提取 API 端点，规范化接口格式，统一发送请求，并将结果以终端表格或本地文件形式输出。

## 核心目标

- 自动化提取单个网页及其关联资源中的 API 接口。
- 识别并规范化不同形式的接口地址。
- 批量发送请求，收集响应状态码、响应长度、响应时间等信息。
- 以表格形式输出测试结果，便于快速筛选异常接口。
- 辅助发现未授权访问、敏感接口暴露、越权访问入口等 API 安全问题。

## 功能规划

### 1. API 自动提取

从目标网页中自动提取可能的 API 端点，包括但不限于：

- HTML 页面中的接口路径。
- JavaScript 文件中的接口地址。
- `fetch`、`XMLHttpRequest`、`axios`、`$.ajax` 等常见请求写法。
- 静态字符串中的相对路径和绝对 URL。
- 常见 RESTful 路由，如 `/api/user/list`、`/v1/order/detail`。
- 带参数的接口，如 `/api/user?id=1`。

### 2. 接口格式识别与规范化

对提取出的接口进行清洗和标准化处理：

- 去重相同接口。
- 将相对路径补全为完整 URL。
- 过滤明显无效的静态资源链接，如图片、字体、样式文件等。
- 识别接口版本前缀，如 `/api/`、`/v1/`、`/v2/`、`/openapi/`。
- 保留查询参数，便于后续请求复现。
- 对缺失协议、缺失域名、重复斜杠等情况进行格式修正。

### 3. 批量请求与响应统计

对整理后的接口统一发起请求，并记录关键信息：

- 请求 URL。
- 请求方法。
- HTTP 状态码。
- 响应长度。
- 响应时间。
- 响应类型。
- 重定向地址。
- 是否疑似可未授权访问。
- 错误信息。

默认情况下可以使用 `GET` 请求进行快速探测，后续可扩展为基于接口特征自动识别 `POST`、`PUT`、`DELETE` 等方法。

### 4. 结果输出

支持将扫描结果输出到终端或保存到本地文件：

- 终端表格输出，适合快速查看。
- CSV 文件，适合导入 Excel 或其他分析工具。
- JSON 文件，适合二次开发或接入其他安全平台。

建议输出字段如下：

| 字段 | 说明 |
| --- | --- |
| `url` | 完整接口地址 |
| `method` | 请求方法 |
| `status_code` | HTTP 状态码 |
| `content_length` | 响应长度 |
| `response_time` | 响应耗时 |
| `content_type` | 响应类型 |
| `redirect_url` | 重定向地址 |
| `risk` | 初步风险标记 |
| `error` | 请求失败原因 |

### 5. 代理支持（可选）

为了适配渗透测试场景，后续可以增加代理功能：

- 支持 HTTP/HTTPS 代理。
- 支持将请求流量转发到 Burp Suite。
- 支持自定义请求头。
- 支持 Cookie 或 Authorization Token 配置。
- 支持忽略 TLS 证书校验。

## 初步风险判断思路

工具本身不应仅凭单一状态码直接判定漏洞，但可以提供辅助风险标记，帮助测试人员优先查看高价值接口。

可作为疑似未授权访问的参考条件：

- 未携带 Cookie 或 Token 时返回 `200`。
- 响应体长度明显大于普通错误页。
- 响应内容为 JSON 且包含业务字段。
- 接口返回用户信息、订单信息、配置数据、权限数据等敏感字段。
- 登录前后响应差异较小。

常见低优先级情况：

- 返回 `401`、`403`。
- 返回统一登录页。
- 返回空响应或固定错误码。
- 返回静态资源或前端路由页面。

## 使用场景

- SRC 漏洞挖掘中的接口资产收集。
- 渗透测试前期的信息收集。
- 前端暴露接口梳理。
- 未授权访问漏洞初筛。
- 敏感接口和高价值 API 快速定位。
- 对目标站点进行接口变更对比和回归检查。

## 预期使用方式

后续工具可以设计为命令行形式，例如：

```bash
python apiextractor.py -u https://example.com
```

指定输出文件：

```bash
python apiextractor.py -u https://example.com -o result.csv
```

指定代理：

```bash
python apiextractor.py -u https://example.com --proxy http://127.0.0.1:8080
```

携带 Cookie：

```bash
python apiextractor.py -u https://example.com --cookie "SESSION=xxxx"
```

忽略 HTTPS 证书校验：

```bash
python apiextractor.py -u https://example.com --insecure
```

## 参数规划

| 参数 | 说明 |
| --- | --- |
| `-u, --url` | 目标网页 URL |
| `-o, --output` | 输出文件路径 |
| `--format` | 输出格式，支持 `table`、`csv`、`json` |
| `--proxy` | HTTP/HTTPS 代理地址 |
| `--cookie` | 自定义 Cookie |
| `-H, --header` | 自定义请求头 |
| `--timeout` | 请求超时时间 |
| `--threads` | 并发请求数量 |
| `--insecure` | 忽略 HTTPS 证书校验 |
| `--include-js` | 是否解析外部 JavaScript 文件 |
| `--same-origin` | 仅测试同源接口 |

- [ ] 对接口进行去重和 URL 标准化。
- [ ] 使用 `GET` 请求批量探测接口。

## 合法使用声明

本项目仅用于授权范围内的安全测试、企业内部资产检查、SRC 漏洞挖掘和学习研究。请勿在未获得授权的情况下对第三方系统进行扫描、测试或攻击。使用者应自行承担因非法使用造成的全部后果。
